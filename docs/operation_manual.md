## 1. 初期セットアップ

### 1.1 環境準備

以下がインストールされていることを確認する。

- Python 3.12
- PostgreSQL 15
- PgAdmin4

### 1.2 Pythonパッケージのインストール

```bash
cd calc_ecl
pip install -r requirements.txt
```

必要なパッケージ: `psycopg2-binary`, `pandas`, `statsmodels`, `scikit-learn`, `flet`, `matplotlib`, `seaborn`

### 1.3 DB接続設定

1. テンプレートをコピーする
   ```bash
   cp config/db.ini.example config/db.ini
   ```
2. `config/db.ini` を開き、接続情報を環境に合わせて編集する
   ```ini
   [postgresql]
   host = localhost
   port = 5432
   dbname = Craft
   schema = calc_ecl
   user = postgres
   password = （実際のパスワード）
   ```

### 1.4 データベース・スキーマ作成

1. PgAdmin4を開く
2. データベース `Craft` を新規作成する（既に存在する場合はスキップ）
3. クエリツールで `CREATE SCHEMA IF NOT EXISTS calc_ecl;` を実行する
4. `db/migrations/001_create_tables.sql` の内容をクエリツールにコピー＆ペーストして実行する
5. `db/migrations/002_create_target_tables.sql` の内容をクエリツールにコピー＆ペーストして実行する
6. `db/migrations/003_add_fiscal_year_month.sql` の内容をクエリツールにコピー＆ペーストして実行する
7. `db/migrations/004_create_ecl_tables.sql` の内容をクエリツールにコピー＆ペーストして実行する
8. 12テーブルが `calc_ecl` スキーマ内に作成されたことを確認する
9. 既存データがある場合: `db/migrations/005_update_reference_date_to_period_end.sql` を実行し、`reference_date` を期末日に変換する（新規環境では不要）

作成されるテーブル一覧:

| 1 | indicator_sources | データソース |
| 2 | indicator_definitions | 指標定義マスタ |
| 3 | indicator_datasets | データセット（取得回） |
| 4 | indicator_data | 指標データ本体（説明変数） |

| 10 | target_data | 目的変数データ本体（PD/LGD等） |

## 2. 指標データのインポート

### 2.1 CSVの取得

1. [統計ダッシュボード](https://dashboard.e-stat.go.jp/) にアクセスする
2. 必要な指標を選択してCSVをダウンロードする
3. ダウンロードしたCSVを `indicator/` フォルダに配置する

### 2.2 インポート実行（CLIの場合）

```bash
python src/import_indicators.py indicator/DataSearchResult_XXXXXX.csv \
    --retrieved-at 2026-03-04 \
    --fiscal-year-month 2026-03-01
```

| 引数 | 説明 | 必須 |
|------|------|------|
| CSVパス | CSVファイルのパス | o |
| `--retrieved-at` | データ取得日（YYYY-MM-DD） | 省略時は当日 |
| `--fiscal-year-month` | 決算年月（YYYY-MM-DD、月初日） | 省略可 |

### 2.2.1 GUIからのインポート

GUIアプリの「データ閲覧」タブからもインポートできる:

1. 「CSVインポート」ボタンを押す
2. **決算年と決算月を選択する**（ドロップダウンで決算年・決算月を指定）
3. ファイル選択ダイアログでCSVファイルを選択する
4. 同じ決算年月のデータセットが既に存在する場合、**上書き確認ダイアログ**が表示される
   - 「はい」を選択すると、既存データを削除して新しいCSVデータで置き換える
   - 「いいえ」を選択するとインポートをキャンセルする
5. 未知のカラムがあれば一括でダイアログ表示される（後述）
6. インポート完了後、データセット一覧が自動更新される

**注意**: データセットのドロップダウンには決算年月の情報が表示される（例: `2026年3月取得 [2026年3月期]`）。

### 2.3 未知のカラムが見つかった場合

CSVに新しい指標カラム（基準年改定で名前が変わったもの等）が含まれていると、未知のカラムが**一括でまとめて**表示される。

#### CLIの場合

```
============================================================
未登録のカラム: 「国内総生産（支出側）（名目）2025年基準【10億円】」
============================================================
  自動検出 - 指標名: 国内総生産（支出側）（名目）2025年基準
  自動検出 - 基準年: 2025
  自動検出 - 単位:   10億円
  自動検出 - 粒度:   quarterly

  (a)追加 / (s)スキップ登録 / (i)今回だけ無視 ? [a/s/i]:
```

#### GUIの場合

複数の未知カラムがある場合、1つずつダイアログを表示するのではなく、**全ての未知カラムを1つのダイアログにまとめて一括表示**する。各カラムについて「追加」「スキップ」「無視」を選択できる。

以下の4項目は**カラム名とCSVデータから自動検出**される:
- **指標名**: カラム名から単位部分を除去
- **基準年**: カラム名の「YYYY年基準」から抽出
- **単位**: カラム名の「【単位】」から抽出
- **粒度**: CSVデータをスキャンし、値がある行の時点パターンから自動判定

**手動入力が必要なのは指標コードのみ**:
- **指標コード**: 英語snake_case（例: `gdp_nominal_2025base`）

入力内容は `config/column_mapping.json` に自動保存され、次回以降は自動認識される。

### 2.4 実行結果の確認

正常に完了すると以下のように表示される:

```
CSV読み込み完了: 555行
マッピング済み指標: ['total_population', 'unemployment_rate', 'ci_index_2020base', ...]
データソースID: 1
指標定義マスタ登録完了
データセットID: 1 (2026-03-04取得)

完了: 520件INSERT, 35件スキップ
```

スキップされる行は、時点が解析できない行や指標値が空の行。

### 2.4 PgAdmin4での確認

インポート後、以下のSQLで内容を確認できる。

```sql
-- データセット一覧
SELECT * FROM indicator_datasets ORDER BY dataset_id DESC;

-- 月次データの確認（直近12ヶ月）
SELECT reference_date, indicators
FROM indicator_data
WHERE frequency = 'monthly' AND region_code = '00000'
ORDER BY reference_date DESC
LIMIT 12;

-- 特定の指標値を取り出す（例: 失業率）
SELECT reference_date,
       indicators->>'unemployment_rate' AS 失業率
FROM indicator_data
WHERE frequency = 'monthly' AND region_code = '00000'
ORDER BY reference_date DESC
LIMIT 12;
```

## 3. 目的変数データのインポート

目的変数（PD: デフォルト率、LGD: 損失率）のデータをCSVからインポートする。

### 3.1 CSVフォーマット

以下の形式でCSVファイルを作成する。

```csv
時点,セグメントコード,セグメント名,pd_corporate,lgd_corporate
2020年度,corporate,法人,0.0234,0.45
2021年度,corporate,法人,0.0198,0.42
2022年度,corporate,法人,0.0212,0.43
```

| 列名 | 説明 | 必須 |
|------|------|------|
| 時点 | 基準日（例: `2020年度`, `2023年1月`, `2022年1-3月期`） | o |
| セグメントコード | セグメント区分コード（例: `corporate`） | 省略時は `all` |
| セグメント名 | セグメント名（例: `法人`） | 省略時は `全体` |
| （それ以外の列） | 目的変数データ（列名 = target_code） | 1列以上 |

**ポイント**:
- 時点列の形式は説明変数CSVと同じ（`parse_time_point()` で解析）
- セグメント列は省略可能（省略時は全体データとして扱う）
- 数値列の列名がそのまま目的変数コード（target_code）になる

### 3.2 CLIでのインポート

```bash
python src/import_targets.py target_data/pd_lgd_2026.csv \
    --dataset-name "2026年3月 法人PD/LGD実績" \
    --retrieved-at 2026-03-04
```

| 引数 | 説明 | 必須 |
|------|------|------|
| CSVパス | CSVファイルのパス | o |
| `--dataset-name` | データセット名 | o |
| `--retrieved-at` | データ取得日（YYYY-MM-DD） | 省略時は当日 |

### 3.3 未登録の目的変数コードが見つかった場合

CSVに新しい目的変数コード（DBに未登録）が含まれていると、以下の情報入力を求められる:

```
============================================================
未登録の目的変数コード: pd_corporate
============================================================
  日本語名を入力 (例: 法人PD): 法人PD
  目的変数タイプ (pd/lgd/ead): pd
  単位 (例: %, 円): %
  frequency (monthly/quarterly/calendar_year/fiscal_year): fiscal_year
```

入力内容は `target_definitions` テーブルに自動登録される。

### 3.4 GUIからのインポート

GUIアプリの「データ閲覧」→「目的変数データ」タブからもインポートできる:

1. 「目的変数CSVインポート」ボタンを押す
2. ファイル選択ダイアログでCSVファイルを選択する
3. データセット名を入力する
4. 未登録の目的変数コードがあれば、日本語名・タイプ・単位の入力ダイアログが表示される
5. 「インポート実行」を押す

### 3.5 PgAdmin4での確認

```sql
-- 目的変数データセット一覧
SELECT * FROM target_datasets ORDER BY target_dataset_id DESC;

-- 目的変数定義一覧
SELECT * FROM target_definitions ORDER BY target_id;

-- データ確認（年度データ）
SELECT reference_date, segment_code, segment_name, targets
FROM target_data
WHERE frequency = 'fiscal_year'
ORDER BY reference_date DESC
LIMIT 12;

-- 特定の目的変数を取り出す（例: 法人PD）
SELECT reference_date, segment_code,
       targets->>'pd_corporate' AS 法人PD
FROM target_data
WHERE frequency = 'fiscal_year' AND segment_code = 'corporate'
ORDER BY reference_date DESC;
```

### 4.3 データ閲覧タブのサブタブ構成

データ閲覧タブは3つのサブタブで構成される（上部のボタンで切替）。

| ボタン | 機能 |
|--------|------|
| ① 説明変数データ | 説明変数（マクロ経済指標）の表示・プロット・インポート |
| ② 目的変数データ | 目的変数（PD/LGD/EAD）の表示・プロット・インポート |
| ③ 比較プロット | 説明変数と目的変数を1グラフに重ねて視覚的に比較する |

#### ③ 比較プロットの使い方

1. まず「① 説明変数データ」タブでデータセットを選択する（自動ロード）
2. 「② 目的変数データ」タブでデータセットを選択する（自動ロード）
3. 「③ 比較プロット」タブに切替
4. 「変数リストを更新」ボタンを押す→説明変数・目的変数のチェックリストが表示される
5. 比較したい変数にチェックを入れる
6. オプションを設定する:
   - **0-1正規化**: スケールが大きく異なる変数を同じ軸で比較したい場合にON
   - **2軸プロット**: 目的変数（PD/LGD）を右軸に表示したい場合にON
7. 「比較プロット実行」を押す

> **使いどころ**: 説明変数の変化が目的変数（PD/LGD）より数期先行するか目視で確認できる。相関分析の前の事前確認に有効。

#### 変換してプロット

①・②タブのプロットエリアに「変換方法」ドロップダウンと「変換してプロット」ボタンがある。
変換後の形を確認してからARIMAや回帰分析に適用する変換方法を決める。

### 4.4 基本操作フロー

1. **データ閲覧タブ（説明変数）**: データセットとfrequencyを選択→指標テーブルが表示される
2. **データ閲覧タブ（目的変数）**: 目的変数データセットを選択→PD/LGDテーブルが表示される
3. **分析タブへ移動**: 各分析タブ（②〜⑤）の上部に「データソース選択」エリアが表示される
4. **データソースを選択**: 説明変数・目的変数それぞれのデータセットとfrequencyを選択する
   - **重要**: 目的変数と説明変数のfrequencyを揃える（例: 両方とも`fiscal_year`）
   - 目的変数のfrequency変更時、説明変数のfrequencyも自動的に合わせられる
   - frequencyが異なる場合は警告メッセージが表示される
5. **目的変数・説明変数を選択**: 変数セレクタで分析対象の変数を選び、分析実行ボタンを押す
6. **結果の確認**: グラフとテーブルで結果が表示される

### 4.4.1 回帰分析タブ — 係数テーブルの VIF 列の読み方

③ 回帰分析タブの係数テーブルには、各説明変数の**VIF（分散拡大係数）**列が追加されている。

| VIF値 | 表示色 | 意味と対処 |
|-------|--------|-----------|
| VIF ≤ 5 | 黒（通常） | 多重共線性なし。問題なし |
| 5 < VIF ≤ 10 | オレンジ | 要注意。相関の強い変数を特定し、どちらか一方の除外を検討する |
| VIF > 10 | 赤 | 多重共線性あり。変数の除外またはPCAでの次元削減を強く推奨 |
| `-`（ハイフン） | — | 切片（Intercept）にはVIFは計算されないため表示なし |

> **VIFとは**: ある説明変数が他の説明変数でどれだけ説明できるかを示す指標。VIF = 1/(1-R²)で計算し、値が大きいほど他の変数と強く相関している（多重共線性が高い）ことを意味する。

### 4.4.2 動的回帰タブ — 標準化トグルの自動判定

⑤ 動的回帰タブで変数を選択すると、各変数の**標準化トグルが自動的に判定**される。

| 状態 | 判定条件 | 意味 |
|------|---------|------|
| 「標準化」トグル（有効・操作可） | 変数に0〜1の範囲外の値が存在する | 標準化が有効。必要に応じてON/OFFを切り替える |
| 「標準化(不要)」トグル（グレーアウト・操作不可） | 変数の全値が[0,1]の範囲内 | PD/LGDなど比率系変数を自動検出。標準化すると解釈が困難になるため無効化 |

> **なぜ[0,1]変数を自動検出するか**: PD（デフォルト率）・LGD（損失率）などの率変数はすでに0〜1のスケールに収まっており、標準化（z-score）は不要かつ解釈を困難にする。自動検出によりこれらの変数への誤った標準化を防止する。

### 4.5 データ変換の選択肢

| 変換方法 | 説明 | 使いどころ |
|---------|------|------------|
| 変換なし | 原系列そのまま | デフォルト |
| 対数変換 | ln(x) | 値が大きくばらつきが大きい場合 |
| 差分化 | x(t) - x(t-1) | トレンド除去・定常化 |
| 対数差分 | 対数変換後に差分化 | 成長率・変化率に変換 |
| 逆双曲線正弦 | arcsinh(x) | 0や負の値も含むデータ |
| 逆双曲線正弦差分 | arcsinh変換後に差分化 | 上記の組み合わせ |

## 5. 年次運用フロー

毎年度の引当金計算に向けた作業フロー。

### 5.1 年次作業スケジュール

```
1. 統計ダッシュボードから最新の説明変数データを取得（CSV）
       ↓
2. 説明変数CSVをインポート（決算年月を指定して登録。同一決算年月は上書き）
       ↓
3. 目的変数（PD/LGD実績）のCSVを準備する
       ↓
4. 目的変数CSVをインポート（CLI or GUIから）
       ↓
5. GUIアプリを起動
       ↓
6. データ閲覧タブで説明変数・目的変数のデータを確認
   └─ ③比較プロットで説明変数と目的変数の目視確認（先行・遅延の有無）
       ↓
7. 相関分析タブで変数の関係性を確認（VIFチェック）
       ↓
8. モデル選択タブで説明変数の最適組み合わせを探索
       ↓
9. 回帰分析タブ or 動的回帰タブでモデルを確認
       ↓
10. モデル確定タブで回帰モデルをDBに登録
       ↓
11. ARIMAタブで説明変数ごとに時系列予測を実施し、予測結果をDBに保存
       ↓
12. ECL計算タブで モデル + ARIMA予測 + シナリオウェイトを設定してECLを算出
       ↓
13. 結果をDBに保存 or CSVエクスポート
```

### 5.2 基準年が改定された場合

基準年の改定（例: GDP 2020年基準 → 2025年基準）が行われた場合:

1. 統計ダッシュボードから新基準年のCSVをダウンロードする
2. `python src/import_indicators.py` を実行する
3. カラム名が変わっていれば未知カラムとして検出され、対話プロンプトが表示される
4. 新しい指標コード（例: `gdp_nominal_2025base`）を入力する
5. **旧データセットは削除しない**（過去モデルの再現に必要）

コードの修正は不要。マッピングは `config/column_mapping.json` に自動保存される。

### 5.3 指標を追加・変更する場合

モデルのフィッティングに合わせて説明変数を変更する場合:

1. 統計ダッシュボードから新しい指標を含むCSVを取得する
2. `python src/import_indicators.py` を実行する
3. 未知カラムの対話プロンプトで指標情報を入力する
4. 新しいデータセットとしてインポートされる

## 6. ARIMAページの操作手順

### 6.1 ARIMAとは

**ARIMA（自己回帰和分移動平均）モデル**は、過去の時系列データのパターンを学習し、将来の値を予測する手法。ECL計算では「説明変数（GDP成長率・失業率等）の将来値」を生成するために使用する。

> **このページの年次運用での役割**: 毎年、各説明変数について最新データでARIMAを再推定し、3〜5年分の予測値を生成する。その予測値を回帰モデルに代入することで、PD/LGDの将来予測値が計算される。

### 6.2 ADF検定（定常性検定）の手順と結果の読み方

ADF検定は「このデータは安定した変動をしているか（定常か）」を判定する。ARIMAモデルの差分次数 d を決めるために実施する。

#### 操作手順

1. 対象変数ドロップダウンで分析したい指標を選ぶ
2. 「ADF検定 + ACF/PACF」ボタンを押す
3. ADF検定結果（ADF統計量・p値・定常性判定）が表示される

#### 結果の読み方

| p値 | 判定 | 次数 d の決め方 |
|-----|------|--------------|
| **p < 0.05** | 定常（単位根なし） | **d = 0**（差分不要） |
| **0.05 ≤ p < 0.1** | 境界領域 | 1次差分後に再検定を推奨 |
| **p ≥ 0.1** | 非定常（トレンドあり） | **d = 1**（1回差分） |

```
例:
  系列: 完全失業率（2000〜2025年）
  ADF統計量: -3.21 / p値: 0.021

  → p値 = 0.021 < 0.05 → 定常 → d=0 でARIMAを構築できる

例（非定常の場合）:
  系列: 名目GDP
  ADF統計量: -0.85 / p値: 0.812

  → p値 = 0.812 >> 0.05 → 非定常（トレンドあり）
  → 対数差分変換（= 成長率）に変換してから再度ADF検定を行う
```

> **実務的な目安**: GDPや物価指数など「右肩上がりのトレンドがある指標」は非定常になりやすい。差分変換（成長率）に変換してから検定するのが通常のフロー。

### 6.3 ACF/PACFグラフの見方と次数(p, d, q)の選び方

ADF検定の下にACF（自己相関）とPACF（偏自己相関）グラフが表示される。

#### グラフの読み方

| 項目 | 意味 |
|------|------|
| **横軸（ラグ数）** | 何期前との相関を見ているか |
| **縦軸（相関係数）** | −1〜+1。0から離れるほど相関が強い |
| **青い点線（信頼区間）** | 点線の外に棒が出ている = 統計的に有意な自己相関がある |

#### ACF/PACFから次数を読む

| グラフの形状 | 示唆される次数 |
|------------|-------------|
| ACFがラグq以降で急激に0になる / PACFが徐々に減衰 | **q = ACFがカットオフするラグ数**（MA成分）|
| PACFがラグp以降で急激に0になる / ACFが徐々に減衰 | **p = PACFがカットオフするラグ数**（AR成分）|
| ACF・PACFともに徐々に減衰 | まだ非定常の可能性 → 差分を再確認 |

> **困ったときは「自動選択」を活用する**: グラフの解釈が難しい場合は、「次数自動選択」機能（AIC最小モデルを自動探索）を使用することを推奨する。

#### 実務での典型的なパターン

| 系列の性質 | 多い次数 |
|----------|---------|
| マクロ経済指標（GDP・物価等） | ARIMA(0,1,1) または ARIMA(1,1,0) |
| 失業率・金利など水準が安定 | ARIMA(1,0,0) または ARIMA(0,0,1) |
| 季節性のある月次データ | 季節ARIMAの検討が必要（本システムでは非対応） |

### 6.4 ARIMA推定の実行と結果の確認

#### 手動次数指定での実行

1. ADF検定の結果を参考に p・d・q を入力する
2. 予測期間（例: 3）を入力する
3. 「ARIMA学習」ボタンを押す

#### 次数自動選択での実行（推奨）

1. max_p・max_d・max_q を指定する（例: 各3）
2. 「自動選択実行」ボタンを押す
3. 上位10モデル一覧が表示される。AIC最小のモデルを推奨次数として採用する

#### 結果の確認ポイント

| 確認項目 | 良好な状態 |
|---------|----------|
| AICが十分小さいか | 同一系列内で他の次数より小さいことを確認 |
| 「converged = True」か | False の場合はそのモデルは使用不可 |
| 係数のp値 | 0.05未満が理想（ただし d=1 後の系列では緩めに判断してよい） |

### 6.5 予測の実行と予測区間の解釈

1. 次数指定 or 自動選択でモデルを確定させる
2. 予測期間（例: 3）を入力して「ARIMA学習」または「自動選択実行」を押す
3. 将来予測グラフと予測値テーブルが表示される

#### 予測テーブルの見方

| 列名 | 意味 | ECLへの活用 |
|------|------|-----------|
| **forecast（予測値）** | モデルが最も可能性が高いと判断した予測値 | **ベースシナリオ**の説明変数値として使用 |
| **lower（下限）** | 95%信頼区間の下限（楽観的な値） | **楽観シナリオ**の参考値 |
| **upper（上限）** | 95%信頼区間の上限（悲観的な値） | **悲観シナリオ**の参考値 |

> **注意**: 変数によってlower/upperのシナリオ対応が逆になる場合がある。
> - 失業率: lower（低失業率）= 楽観、upper（高失業率）= 悲観 → **正しい**
> - GDP成長率: lower（低成長）= 悲観、upper（高成長）= 楽観 → **逆になるため注意**

> **予測期間が長くなるほど信頼区間は広がる**: 3年超の予測は不確実性が高い。ECL計算で使用する際はその点を認識し、保守的なシナリオウェイトの設定を検討する。

### 6.6 予測結果のDB保存

1. 予測実行後、「予測結果をDBに保存」エリアが表示される
2. **対象決算年月**: モデルのデータセットに紐づく `fiscal_year_month` が自動表示される（設定不要）
3. メモ（任意）を入力して「DBに保存」ボタンを押す
4. 保存後、ページ下部の「保存済みARIMA予測（このデータセット）」一覧が自動更新される

> **注意**: ページを開いた時点でデータセットに紐づく全変数の保存済み予測が一覧表示される。不要な予測は削除ボタンで削除できる。

### 6.7 よくあるトラブルと対処方法

| 症状 | 原因 | 対処 |
|------|------|------|
| ADF検定で「定常」と判定されたが予測値が発散する | 過学習またはモデルが不安定 | p・d・qを小さい値（例: 各1）に絞って再試行する |
| 自動選択で「全モデル失敗」と表示される | データが短すぎる（15未満）or 欠損あり | データの長さと欠損値を確認。max_p・max_qを1〜2に下げる |
| 予測値がマイナスになる（比率系変数で問題） | ARIMAは制約なしで予測するため負の値が出うる | 変換変数（差分化後）で予測し、元の単位に逆変換して使用することを検討する |
| nlagsエラー（データ数不足） | デフォルトのnlagsがデータ数に対して大きすぎる | nlags入力フィールドに「データ数÷3」程度の値を手動入力する（例: データ数30→10） |

## 7. モデル確定タブの操作手順

### 7.1 モデル確定タブの役割

回帰分析タブまたは動的回帰タブで決定した「最良モデル」をDBに登録する。登録したモデルはECL計算タブで参照される。

> **年次運用での役割**: モデル（説明変数の組み合わせ・変換方法・係数）が確定したら、このタブで保存する。翌年以降もモデルを引き継ぐ場合は再保存不要。変更時のみ新規登録する。

### 7.2 モデルの登録手順

1. データソース選択エリアでデータセット・frequencyを選択する
2. 目的変数・説明変数を選択する
3. 変換方法・標準化・ラグ数を設定する（回帰分析タブと同じ設定を使うこと）
4. 「回帰を実行して確認」ボタンを押す
5. 回帰結果（Adj.R² / AIC / DW / 係数テーブル）が表示されることを確認する
6. モデル名とメモを入力する
7. 「このモデルをDBに保存」ボタンを押す

### 7.3 保存済みモデルの管理

- タブ下部に保存済みモデル一覧が表示される
- 不要なモデルは「削除」ボタンで削除できる（ECL計算タブで選択中のモデルは削除しないこと）
- モデルID（config_id）がECL計算タブでの参照に使われる

## 8. ECL計算タブの操作手順

### 8.1 ECL計算タブの役割

IFRS9に基づく予想信用損失（ECL）を3シナリオ加重平均で計算する。

**必要な事前準備**:
- ⑧ モデル確定タブでモデルを保存済みであること
- ⑥ ARIMAタブで各説明変数の予測結果をDBに保存済みであること

### 8.2 Step 1: モデルの選択

1. 「回帰モデルを選択」ドロップダウンから使用するモデルを選ぶ
2. モデルが選択されると、そのモデルで使用している説明変数が一覧表示される

### 8.3 Step 2: ARIMA予測の割り当て

各説明変数について、どのARIMA予測結果を使うかを割り当てる。

1. 各変数の横に「ARIMA予測を選択」ドロップダウンが表示される
2. ドロップダウンには同じ変数コードのARIMA予測が一覧表示される
3. 使用するARIMA予測（forecast_id）を選択する

> **注意**: 変数によってlower/upperのシナリオ対応が逆になる。
> - 失業率: lower（低失業率）= 楽観シナリオ
> - GDP成長率: lower（低成長）= 悲観シナリオ（逆になるため注意）
>
> 現バージョンでは `forecast` → ベース / `lower` → 楽観 / `upper` → 悲観 に固定されている。変数の性質に応じてシナリオウェイトで調整すること。

### 8.4 Step 3: シナリオウェイトと LGD/EAD の設定

| 項目 | 説明 | 合計 |
|------|------|------|
| ベースシナリオウェイト | 最も可能性が高い経済シナリオの重み | 合計が1.0（100%）になること |
| 楽観シナリオウェイト | 景気が予想より良かった場合の重み | |
| 悲観シナリオウェイト | 景気が予想より悪かった場合の重み | |
| LGD（損失率） | デフォルト時の損失割合（例: 0.45 = 45%） | |
| EAD（エクスポージャー） | 引当計算の基準となる与信残高（円） | |

> **典型的なウェイト例**:
> - ベース 50% / 楽観 25% / 悲観 25%
> - ベース 60% / 楽観 20% / 悲観 20%

### 8.5 ECLの計算と結果確認

1. 「ECLを計算する」ボタンを押す
2. 計算結果が表示される:
   - **シナリオ別グラフ**: ベース/楽観/悲観/加重平均の4系列を時系列プロット
   - **結果テーブル**: 期間ごとのPD・ECLレート・ECL金額

#### 結果の見方

| 項目 | 意味 |
|------|------|
| pd_weighted | 3シナリオの加重平均PD |
| ecl_rate | `pd_weighted × LGD` = ECLレート |
| ecl_amount | `ecl_rate × EAD` = ECL引当金額（円） |

### 8.6 結果の保存・出力

- **「結果をDBに保存」ボタン**: ecl_results テーブルに保存する。**決算年月はモデルのデータセットから自動設定される**（手動入力不要）
- **「CSVエクスポート（クリップボード）」ボタン**: 計算結果をCSV形式でクリップボードにコピーする（macOS: pbcopy / Windows: clip を使用）。テキストエディタやExcelに貼り付けて保存すること

### 8.7 保存済みECL計算結果の確認

ページ下部の「保存済みECL計算結果」エリアに保存済みの一覧が常時表示される。

| 操作 | 説明 |
|------|------|
| **決算年月フィルター** | ドロップダウンで特定の決算年月だけに絞り込める。「（すべて）」で全件表示 |
| **更新ボタン（🔄）** | 一覧を手動で再読み込みする |
| **詳細表示ボタン（表アイコン）** | その行のECL計算結果テーブル（期間別PD・ECL率・ECL金額）をダイアログで表示する |
| **削除ボタン（🗑）** | その行のECL結果をDBから削除する |

## 9. トラブルシューティング

### 9.1 説明変数インポート時のエラー

| エラーメッセージ | 原因 | 対処 |
|----------------|------|------|
| `設定ファイルが見つかりません` | `config/db.ini` が未作成 | `db.ini.example` をコピーして作成 |
| `connection refused` | PostgreSQLが起動していない | PostgreSQLサービスを起動 |
| `UNIQUE constraint` | 同じデータセット・日付・地域の重複 | 既にインポート済み。新規データセットとしてインポートする |
| `UnicodeDecodeError` | CSVの文字コード | UTF-8で保存し直す |

### 9.2 目的変数インポート時のエラー

| エラーメッセージ | 原因 | 対処 |
|----------------|------|------|
| `時点列が見つかりません` | CSVに「時点」列がない | 1列目を「時点」に変更する |
| `数値カラムが見つかりません` | データ列がすべて非数値 | 数値データ列が含まれているか確認する |
| `UNIQUE constraint` | 同じデータセット・日付・セグメントの重複 | 既存データはJSONBマージされる。意図しない場合は新データセットとしてインポートする |
| `invalid input value for enum target_type` | target_typeに無効な値 | `pd`, `lgd`, `ead` のいずれかを指定する |

### 9.3 PgAdmin4でのSQL実行エラー

- DDL実行時にエラーが出た場合、エラーが出た文だけが失敗し、それ以前のテーブルは作成される
- 既にテーブルが存在する場合は `relation "xxx" already exists` エラーになる。問題がなければ無視してよい

### 9.4 GUIアプリのエラー

| エラーメッセージ | 原因 | 対処 |
|----------------|------|------|
| `先にデータ閲覧タブでデータを読み込んでください` | 他タブに先に移動した | まずデータ閲覧タブでデータセットを選択する |
| `目的変数と説明変数の期間が重なりません` | frequencyが異なる（例: 説明変数=monthly, 目的変数=fiscal_year） | 分析タブ上部の「データソース選択」でfrequencyを揃える |
| `データセット取得エラー` | DB接続失敗 | `config/db.ini` の設定を確認、PostgreSQL起動確認 |
| `プロットエラー` | データ変換でNaN/Infが発生 | 変換方法を変更する（0や負の値に対数変換は不可） |
