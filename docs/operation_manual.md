## 1. 初期セットアップ

### 1.1 環境準備

以下がインストールされていることを確認する。

- Python 3.12
- PostgreSQL 15
- PgAdmin4

### 1.2 Pythonパッケージのインストール

```bash
cd calc_ecl
pip install -r requirements.txt
```

必要なパッケージ: `psycopg2-binary`, `pandas`, `statsmodels`, `scikit-learn`, `flet`, `matplotlib`, `seaborn`

### 1.3 DB接続設定

1. テンプレートをコピーする
   ```bash
   cp config/db.ini.example config/db.ini
   ```
2. `config/db.ini` を開き、接続情報を環境に合わせて編集する
   ```ini
   [postgresql]
   host = localhost
   port = 5432
   dbname = Craft
   schema = calc_ecl
   user = postgres
   password = （実際のパスワード）
   ```

### 1.4 データベース・スキーマ作成

1. PgAdmin4を開く
2. データベース `Craft` を新規作成する（既に存在する場合はスキップ）
3. クエリツールで `CREATE SCHEMA IF NOT EXISTS calc_ecl;` を実行する
4. `db/migrations/001_create_tables.sql` の内容をクエリツールにコピー＆ペーストして実行する
5. `db/migrations/002_create_target_tables.sql` の内容をクエリツールにコピー＆ペーストして実行する
6. `db/migrations/003_add_fiscal_year_month.sql` の内容をクエリツールにコピー＆ペーストして実行する
7. 10テーブルが `calc_ecl` スキーマ内に作成されたことを確認する

作成されるテーブル一覧:

| # | テーブル名 | 概要 |
|---|-----------|------|
| 1 | indicator_sources | データソース |
| 2 | indicator_definitions | 指標定義マスタ |
| 3 | indicator_datasets | データセット（取得回） |
| 4 | indicator_data | 指標データ本体（説明変数） |
| 5 | model_configs | モデル設定 |
| 6 | model_results | モデル実行結果 |
| 7 | forecast_scenarios | 将来シナリオ・予測値 |
| 8 | target_definitions | 目的変数定義マスタ |
| 9 | target_datasets | 目的変数データセット（取得回） |
| 10 | target_data | 目的変数データ本体（PD/LGD等） |

## 2. 指標データのインポート

### 2.1 CSVの取得

1. [統計ダッシュボード](https://dashboard.e-stat.go.jp/) にアクセスする
2. 必要な指標を選択してCSVをダウンロードする
3. ダウンロードしたCSVを `indicator/` フォルダに配置する

### 2.2 インポート実行（CLIの場合）

```bash
python src/import_indicators.py indicator/DataSearchResult_XXXXXX.csv \
    --retrieved-at 2026-03-04 \
    --fiscal-year-month 2026-03-01
```

| 引数 | 説明 | 必須 |
|------|------|------|
| CSVパス | CSVファイルのパス | o |
| `--retrieved-at` | データ取得日（YYYY-MM-DD） | 省略時は当日 |
| `--fiscal-year-month` | 決算年月（YYYY-MM-DD、月初日） | 省略可 |

### 2.2.1 GUIからのインポート

GUIアプリの「データ閲覧」タブからもインポートできる:

1. 「CSVインポート」ボタンを押す
2. **決算年と決算月を選択する**（ドロップダウンで決算年・決算月を指定）
3. ファイル選択ダイアログでCSVファイルを選択する
4. 同じ決算年月のデータセットが既に存在する場合、**上書き確認ダイアログ**が表示される
   - 「はい」を選択すると、既存データを削除して新しいCSVデータで置き換える
   - 「いいえ」を選択するとインポートをキャンセルする
5. 未知のカラムがあれば一括でダイアログ表示される（後述）
6. インポート完了後、データセット一覧が自動更新される

**注意**: データセットのドロップダウンには決算年月の情報が表示される（例: `2026年3月取得 [2026年3月期]`）。

### 2.3 未知のカラムが見つかった場合

CSVに新しい指標カラム（基準年改定で名前が変わったもの等）が含まれていると、未知のカラムが**一括でまとめて**表示される。

#### CLIの場合

```
============================================================
未登録のカラム: 「国内総生産（支出側）（名目）2025年基準【10億円】」
============================================================
  自動検出 - 指標名: 国内総生産（支出側）（名目）2025年基準
  自動検出 - 基準年: 2025
  自動検出 - 単位:   10億円
  自動検出 - 粒度:   quarterly

  (a)追加 / (s)スキップ登録 / (i)今回だけ無視 ? [a/s/i]:
```

#### GUIの場合

複数の未知カラムがある場合、1つずつダイアログを表示するのではなく、**全ての未知カラムを1つのダイアログにまとめて一括表示**する。各カラムについて「追加」「スキップ」「無視」を選択できる。

以下の4項目は**カラム名とCSVデータから自動検出**される:
- **指標名**: カラム名から単位部分を除去
- **基準年**: カラム名の「YYYY年基準」から抽出
- **単位**: カラム名の「【単位】」から抽出
- **粒度**: CSVデータをスキャンし、値がある行の時点パターンから自動判定

**手動入力が必要なのは指標コードのみ**:
- **指標コード**: 英語snake_case（例: `gdp_nominal_2025base`）

入力内容は `config/column_mapping.json` に自動保存され、次回以降は自動認識される。

### 2.4 実行結果の確認

正常に完了すると以下のように表示される:

```
CSV読み込み完了: 555行
マッピング済み指標: ['total_population', 'unemployment_rate', 'ci_index_2020base', ...]
データソースID: 1
指標定義マスタ登録完了
データセットID: 1 (2026-03-04取得)

完了: 520件INSERT, 35件スキップ
```

スキップされる行は、時点が解析できない行や指標値が空の行。

### 2.4 PgAdmin4での確認

インポート後、以下のSQLで内容を確認できる。

```sql
-- データセット一覧
SELECT * FROM indicator_datasets ORDER BY dataset_id DESC;

-- 月次データの確認（直近12ヶ月）
SELECT reference_date, indicators
FROM indicator_data
WHERE frequency = 'monthly' AND region_code = '00000'
ORDER BY reference_date DESC
LIMIT 12;

-- 特定の指標値を取り出す（例: 失業率）
SELECT reference_date,
       indicators->>'unemployment_rate' AS 失業率
FROM indicator_data
WHERE frequency = 'monthly' AND region_code = '00000'
ORDER BY reference_date DESC
LIMIT 12;
```

## 3. 目的変数データのインポート

目的変数（PD: デフォルト率、LGD: 損失率）のデータをCSVからインポートする。

### 3.1 CSVフォーマット

以下の形式でCSVファイルを作成する。

```csv
時点,セグメントコード,セグメント名,pd_corporate,lgd_corporate
2020年度,corporate,法人,0.0234,0.45
2021年度,corporate,法人,0.0198,0.42
2022年度,corporate,法人,0.0212,0.43
```

| 列名 | 説明 | 必須 |
|------|------|------|
| 時点 | 基準日（例: `2020年度`, `2023年1月`, `2022年1-3月期`） | o |
| セグメントコード | セグメント区分コード（例: `corporate`） | 省略時は `all` |
| セグメント名 | セグメント名（例: `法人`） | 省略時は `全体` |
| （それ以外の列） | 目的変数データ（列名 = target_code） | 1列以上 |

**ポイント**:
- 時点列の形式は説明変数CSVと同じ（`parse_time_point()` で解析）
- セグメント列は省略可能（省略時は全体データとして扱う）
- 数値列の列名がそのまま目的変数コード（target_code）になる

### 3.2 CLIでのインポート

```bash
python src/import_targets.py target_data/pd_lgd_2026.csv \
    --dataset-name "2026年3月 法人PD/LGD実績" \
    --retrieved-at 2026-03-04
```

| 引数 | 説明 | 必須 |
|------|------|------|
| CSVパス | CSVファイルのパス | o |
| `--dataset-name` | データセット名 | o |
| `--retrieved-at` | データ取得日（YYYY-MM-DD） | 省略時は当日 |

### 3.3 未登録の目的変数コードが見つかった場合

CSVに新しい目的変数コード（DBに未登録）が含まれていると、以下の情報入力を求められる:

```
============================================================
未登録の目的変数コード: pd_corporate
============================================================
  日本語名を入力 (例: 法人PD): 法人PD
  目的変数タイプ (pd/lgd/ead): pd
  単位 (例: %, 円): %
  frequency (monthly/quarterly/calendar_year/fiscal_year): fiscal_year
```

入力内容は `target_definitions` テーブルに自動登録される。

### 3.4 GUIからのインポート

GUIアプリの「データ閲覧」→「目的変数データ」タブからもインポートできる:

1. 「目的変数CSVインポート」ボタンを押す
2. ファイル選択ダイアログでCSVファイルを選択する
3. データセット名を入力する
4. 未登録の目的変数コードがあれば、日本語名・タイプ・単位の入力ダイアログが表示される
5. 「インポート実行」を押す

### 3.5 PgAdmin4での確認

```sql
-- 目的変数データセット一覧
SELECT * FROM target_datasets ORDER BY target_dataset_id DESC;

-- 目的変数定義一覧
SELECT * FROM target_definitions ORDER BY target_id;

-- データ確認（年度データ）
SELECT reference_date, segment_code, segment_name, targets
FROM target_data
WHERE frequency = 'fiscal_year'
ORDER BY reference_date DESC
LIMIT 12;

-- 特定の目的変数を取り出す（例: 法人PD）
SELECT reference_date, segment_code,
       targets->>'pd_corporate' AS 法人PD
FROM target_data
WHERE frequency = 'fiscal_year' AND segment_code = 'corporate'
ORDER BY reference_date DESC;
```

## 4. GUIアプリの起動と操作

### 4.1 起動方法

```bash
cd calc_ecl
PYTHONPATH=. python main.py
```

アプリが起動し、「ECL将来予想モデル」ウィンドウが表示される。
ウィンドウサイズは自由にリサイズ可能。

### 4.2 タブ構成

| # | タブ | 機能 |
|---|------|------|
| 1 | データ閲覧 | 説明変数・目的変数のデータ表示、変換後プロット、比較プロット、CSVインポート |
| 2 | 相関分析 | 相関行列ヒートマップ・VIF一覧・VIFクロス表 |
| 3 | 回帰分析 | OLS回帰・係数テーブル・残差プロット・交差検証 |
| 4 | モデル選択 | 説明変数の全組み合わせ探索・VIFフィルタ |
| 5 | 動的回帰 | 変数別の変換・標準化・ラグ設定 |
| 6 | ARIMA | ADF検定・ACF/PACF・次数自動選択・将来予測 |
| 7 | 将来シナリオ | シナリオ作成・予測結果保存（実装予定） |

### 4.3 データ閲覧タブのサブタブ構成

データ閲覧タブは3つのサブタブで構成される（上部のボタンで切替）。

| ボタン | 機能 |
|--------|------|
| ① 説明変数データ | 説明変数（マクロ経済指標）の表示・プロット・インポート |
| ② 目的変数データ | 目的変数（PD/LGD/EAD）の表示・プロット・インポート |
| ③ 比較プロット | 説明変数と目的変数を1グラフに重ねて視覚的に比較する |

#### ③ 比較プロットの使い方

1. まず「① 説明変数データ」タブでデータセットを選択する（自動ロード）
2. 「② 目的変数データ」タブでデータセットを選択する（自動ロード）
3. 「③ 比較プロット」タブに切替
4. 「変数リストを更新」ボタンを押す→説明変数・目的変数のチェックリストが表示される
5. 比較したい変数にチェックを入れる
6. オプションを設定する:
   - **0-1正規化**: スケールが大きく異なる変数を同じ軸で比較したい場合にON
   - **2軸プロット**: 目的変数（PD/LGD）を右軸に表示したい場合にON
7. 「比較プロット実行」を押す

> **使いどころ**: 説明変数の変化が目的変数（PD/LGD）より数期先行するか目視で確認できる。相関分析の前の事前確認に有効。

#### 変換してプロット

①・②タブのプロットエリアに「変換方法」ドロップダウンと「変換してプロット」ボタンがある。
変換後の形を確認してからARIMAや回帰分析に適用する変換方法を決める。

### 4.4 基本操作フロー

1. **データ閲覧タブ（説明変数）**: データセットとfrequencyを選択→指標テーブルが表示される
2. **データ閲覧タブ（目的変数）**: 目的変数データセットを選択→PD/LGDテーブルが表示される
3. **分析タブへ移動**: 各分析タブ（②〜⑤）の上部に「データソース選択」エリアが表示される
4. **データソースを選択**: 説明変数・目的変数それぞれのデータセットとfrequencyを選択する
   - **重要**: 目的変数と説明変数のfrequencyを揃える（例: 両方とも`fiscal_year`）
   - 目的変数のfrequency変更時、説明変数のfrequencyも自動的に合わせられる
   - frequencyが異なる場合は警告メッセージが表示される
5. **目的変数・説明変数を選択**: 変数セレクタで分析対象の変数を選び、分析実行ボタンを押す
6. **結果の確認**: グラフとテーブルで結果が表示される

### 4.5 データ変換の選択肢

| 変換方法 | 説明 | 使いどころ |
|---------|------|------------|
| 変換なし | 原系列そのまま | デフォルト |
| 対数変換 | ln(x) | 値が大きくばらつきが大きい場合 |
| 差分化 | x(t) - x(t-1) | トレンド除去・定常化 |
| 対数差分 | 対数変換後に差分化 | 成長率・変化率に変換 |
| 逆双曲線正弦 | arcsinh(x) | 0や負の値も含むデータ |
| 逆双曲線正弦差分 | arcsinh変換後に差分化 | 上記の組み合わせ |

## 5. 年次運用フロー

毎年度の引当金計算に向けた作業フロー。

### 5.1 年次作業スケジュール

```
1. 統計ダッシュボードから最新の説明変数データを取得（CSV）
       ↓
2. 説明変数CSVをインポート（決算年月を指定して登録。同一決算年月は上書き）
       ↓
3. 目的変数（PD/LGD実績）のCSVを準備する
       ↓
4. 目的変数CSVをインポート（CLI or GUIから）
       ↓
5. GUIアプリを起動
       ↓
6. データ閲覧タブで説明変数・目的変数のデータを確認
   └─ ③比較プロットで説明変数と目的変数の目視確認（先行・遅延の有無）
       ↓
7. 相関分析タブで変数の関係性を確認（VIFチェック）
       ↓
8. モデル選択タブで説明変数の最適組み合わせを探索
       ↓
9. 回帰分析タブ or 動的回帰タブでモデルを確定
       ↓
10. ARIMAタブで時系列予測を実施
       ↓
11. 将来シナリオ（ベース/楽観/悲観）を作成
       ↓
12. 加重平均でECL算出
```

### 5.2 基準年が改定された場合

基準年の改定（例: GDP 2020年基準 → 2025年基準）が行われた場合:

1. 統計ダッシュボードから新基準年のCSVをダウンロードする
2. `python src/import_indicators.py` を実行する
3. カラム名が変わっていれば未知カラムとして検出され、対話プロンプトが表示される
4. 新しい指標コード（例: `gdp_nominal_2025base`）を入力する
5. **旧データセットは削除しない**（過去モデルの再現に必要）

コードの修正は不要。マッピングは `config/column_mapping.json` に自動保存される。

### 5.3 指標を追加・変更する場合

モデルのフィッティングに合わせて説明変数を変更する場合:

1. 統計ダッシュボードから新しい指標を含むCSVを取得する
2. `python src/import_indicators.py` を実行する
3. 未知カラムの対話プロンプトで指標情報を入力する
4. 新しいデータセットとしてインポートされる

## 6. トラブルシューティング

### 6.1 説明変数インポート時のエラー

| エラーメッセージ | 原因 | 対処 |
|----------------|------|------|
| `設定ファイルが見つかりません` | `config/db.ini` が未作成 | `db.ini.example` をコピーして作成 |
| `connection refused` | PostgreSQLが起動していない | PostgreSQLサービスを起動 |
| `UNIQUE constraint` | 同じデータセット・日付・地域の重複 | 既にインポート済み。新規データセットとしてインポートする |
| `UnicodeDecodeError` | CSVの文字コード | UTF-8で保存し直す |

### 6.2 目的変数インポート時のエラー

| エラーメッセージ | 原因 | 対処 |
|----------------|------|------|
| `時点列が見つかりません` | CSVに「時点」列がない | 1列目を「時点」に変更する |
| `数値カラムが見つかりません` | データ列がすべて非数値 | 数値データ列が含まれているか確認する |
| `UNIQUE constraint` | 同じデータセット・日付・セグメントの重複 | 既存データはJSONBマージされる。意図しない場合は新データセットとしてインポートする |
| `invalid input value for enum target_type` | target_typeに無効な値 | `pd`, `lgd`, `ead` のいずれかを指定する |

### 6.3 PgAdmin4でのSQL実行エラー

- DDL実行時にエラーが出た場合、エラーが出た文だけが失敗し、それ以前のテーブルは作成される
- 既にテーブルが存在する場合は `relation "xxx" already exists` エラーになる。問題がなければ無視してよい

### 6.4 GUIアプリのエラー

| エラーメッセージ | 原因 | 対処 |
|----------------|------|------|
| `先にデータ閲覧タブでデータを読み込んでください` | 他タブに先に移動した | まずデータ閲覧タブでデータセットを選択する |
| `目的変数と説明変数の期間が重なりません` | frequencyが異なる（例: 説明変数=monthly, 目的変数=fiscal_year） | 分析タブ上部の「データソース選択」でfrequencyを揃える |
| `データセット取得エラー` | DB接続失敗 | `config/db.ini` の設定を確認、PostgreSQL起動確認 |
| `プロットエラー` | データ変換でNaN/Infが発生 | 変換方法を変更する（0や負の値に対数変換は不可） |
