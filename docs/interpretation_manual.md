# ECL将来予想モデル 統計分析 結果解釈マニュアル

実務担当者向けの統計分析結果解釈ガイド。
各タブに表示される指標・グラフの読み方と、ECLモデル構築に向けた判断基準を示す。

---

## はじめに — このマニュアルの使い方

本マニュアルはGUIアプリの各タブで得られる統計出力を、実務担当者が自力で解釈できるよう設計されている。

### 全体的な分析の流れとゴール

```
目的: マクロ経済指標（説明変数）からPD/LGD（目的変数）を予測するモデルを構築し、
      IFRS9に基づくECL（予想信用損失）を算出する

【分析フェーズ】
Step 1【②相関分析タブ】
  └─ 変数間の相関を俯瞰し、有望な説明変数候補を絞り込む

Step 2【④モデル選択タブ】
  └─ 絞り込んだ候補から最適な説明変数の組み合わせを網羅的に探索する

Step 3【③回帰分析タブ / ⑤動的回帰タブ】
  └─ 最良モデルの係数・残差を精査する

【確定フェーズ】
Step 4【⑥ARIMAタブ】
  └─ 各説明変数を時系列予測し、予測結果をDBに保存する

Step 5【⑦モデル確定タブ（実装予定）】
  └─ 精査した回帰モデルの設定をDBに正式登録する

【計算フェーズ】
Step 6【⑧ECL計算タブ（実装予定）】
  └─ 保存済みモデル＋ARIMA予測を組み合わせ、3シナリオでECLを算出する
```

## 1. データ閲覧タブ — データの事前確認

### 1.1 時系列グラフの読み方

分析前に目的変数（PD/LGD）と説明変数候補の時系列グラフを確認する。

| 確認ポイント | 見るべき点 | 次のアクション |
|------------|----------|-------------|
| **トレンド** | 全体的に増加・減少しているか | トレンドがある場合、差分変換を検討 |
| **外れ値** | 急激な跳ね上がり・落ち込みがないか | 特定年度のデータ品質を確認 |
| **欠損期間** | 途中でデータが途切れていないか | 途切れている場合、分析対象期間を短縮 |
| **目的変数との視覚的共変動** | 説明変数が目的変数と似た動きをしているか | 同じ方向に動く変数は正の相関候補 |

### 1.2 データ変換の目的と選択基準

分析を行う前にデータを変換することで、モデルの精度が改善する場合がある。

| 変換方法 | 意味 | いつ使うか |
|---------|------|-----------|
| **変換なし** | 原系列そのまま | まず最初に試す。PDのような比率系変数に向く |
| **対数変換** | `ln(1+x)` | GDP・地価など桁が大きく、右裾が重い変数 |
| **差分化** | `x(t) - x(t-1)` | 長期トレンドがある変数の「変化量」を見たい場合 |
| **対数差分** | 対数→差分 | 「成長率」や「変化率」に変換したい場合 |
| **逆双曲線正弦** | `arcsinh(x)` | 0や負の値を含む変数（対数変換が使えない場合） |
| **逆双曲線正弦差分** | 上記→差分 | 負の値を含む変数の変化量を見たい場合 |

> **ECL実務での目安**: PDやLGDは概ね0〜1の比率系データのため、原系列のまま扱うか対数変換を試すことが多い。GDPや物価指数は対数差分（=成長率）への変換を検討する。

## 2. 相関分析タブ — 変数間の関係性を把握する

### 2.1 相関行列ヒートマップ

#### 出力内容

変数のすべての組み合わせについて**ピアソン相関係数**（−1〜+1）をカラーマップで表示したもの。

#### 読み方

| 色 | 値の範囲 | 意味 |
|----|---------|------|
| 濃い赤 | +0.7 〜 +1.0 | 強い正の相関 |
| 薄い赤 | +0.3 〜 +0.7 | 中程度の正の相関 |
| 白・薄い色 | −0.3 〜 +0.3 | ほぼ無相関 |
| 薄い青 | −0.7 〜 −0.3 | 中程度の負の相関 |
| 濃い青 | −1.0 〜 −0.7 | 強い負の相関 |

対角線は自己相関（常に+1.0）。

#### ECLモデル構築に向けた判断

**目的変数との相関（ヒートマップの1行または1列）を重点的に確認する。**

| 相関係数の絶対値 | 解釈 | アクション |
|--------------|------|-----------|
| 0.7以上 | 強い説明変数候補 | 回帰分析に優先的に採用 |
| 0.3〜0.7 | 中程度の説明変数候補 | 他の変数との組み合わせで採用を検討 |
| 0.3未満 | 説明力が弱い | 除外を検討（ただし理論的根拠があれば残す） |

**説明変数間の相関（多重共線性のスクリーニング）も必ず確認する。**

| 説明変数間の相関係数絶対値 | 解釈 | アクション |
|----------------------|------|-----------|
| 0.9以上 | 非常に強い多重共線性の疑い | **片方を除外する** |
| 0.7〜0.9 | 強い相関。VIF確認が必要 | VIF（後述）で詳細確認 |
| 0.7未満 | 概ね問題なし | そのまま続行 |

> **注意**: 相関係数は線形関係しか捉えられない。非線形の関係（例: 失業率が一定水準を超えると急激にPDが上昇）は相関係数には現れない。

#### 実務での解釈例

```
例: 失業率（unemployment_rate）と法人PD（pd_corporate）の相関係数 = +0.65

解釈:
  - 失業率が上昇すると、法人PDも上昇する傾向がある（正の相関）
  - 相関は中程度〜強い水準であり、有望な説明変数候補といえる
  - 直感的にも「経済悪化 → 失業率上昇 → 企業の資金繰り悪化 → PD上昇」と整合している
```

### 2.2 VIF一覧（分散拡大係数）

#### 出力内容

各説明変数について**VIF（Variance Inflation Factor: 分散拡大係数）**を計算したもの。

#### VIFとは何か

VIFは「ある説明変数が、他の説明変数によってどれだけ説明できるか」を示す指標。
多重共線性（説明変数間の強い相関）を定量的に検出するために使用する。

**計算式**: `VIF = 1 / (1 - R²)`

ここで R² は「その変数を目的変数として、他の全説明変数で回帰したときの決定係数」。

#### 判断基準

| VIF値 | 解釈 | アクション |
|-------|------|-----------|
| 1.0 | 他の変数と全く相関がない（理想的） | そのまま使用 |
| 1〜5 | 多重共線性なし | 問題なし |
| 5〜10 | 多重共線性の兆候 | 注意が必要。係数の符号を確認 |
| **10以上** | **多重共線性あり** | **片方の変数を除外する** |
| ∞（無限大） | 完全な多重共線性 | 必ず除外する |

#### 多重共線性が問題になる理由

多重共線性が存在すると：

1. **係数の推定が不安定になる**: 少しデータが変わると係数が大きく変化する
2. **係数の符号が逆転することがある**: 経済学的に説明できない正負の逆転が起きる
3. **p値が大きくなる**: 本来は有意な変数でも「有意でない」と判定されてしまう

```
例: GDP（gdp_nominal）と鉱工業生産指数（ci_index）のVIF = 15.3

解釈:
  - GDPを他の変数で回帰すると R² ≒ 0.93 程度の高い値になる
  - GDPと鉱工業生産指数はほぼ同じ情報を持っている
  - 両方を同時にモデルに入れると係数が不安定になる
  → どちらか一方（理論的根拠が強い方）を選択して除外する
```

---

### 2.3 VIFクロス表（ペアVIF）

#### 出力内容

全変数ペアについて、**2変数間のVIF（= 1/(1-r²)）**をマトリクス形式で表示したもの。

#### 多変数VIFとの違い

| 指標 | 計算方法 | 用途 |
|------|---------|------|
| **VIF一覧** | 1変数 vs 他の全変数 | モデルに複数変数を入れたとき全体の多重共線性 |
| **VIFクロス表** | 1変数 vs 1変数（ペア） | 2変数の組み合わせの相性を事前スクリーニング |

#### 読み方

- 対角（自分自身）は常に 1.0
- セルの値が 10 以上のペアは「このペアを同時にモデルに入れることは避ける」ことを意味する
- モデル選択タブで組み合わせを絞り込む前の予備チェックとして活用する

## 3. 回帰分析タブ — モデルの精度と係数を確認する

### 3.1 回帰分析の全体像

OLS（最小二乗法）回帰は、目的変数（PD/LGD）を説明変数（マクロ経済指標）の線形結合で説明するモデル。

**モデル式**: `PD(t) = β₀ + β₁×X₁(t-lag) + β₂×X₂(t-lag) + ... + ε(t)`

### 3.2 モデル全体の評価指標

#### R²（決定係数）

| 値 | 解釈 |
|----|------|
| 0〜1の値 | 目的変数の変動のうち、モデルで説明できる割合 |
| 0.8以上 | 当てはまりが良い |
| 0.5〜0.8 | 中程度 |
| 0.5未満 | 当てはまりが弱い（変数の選び直し・変換を検討） |

> **注意**: R²は説明変数を増やすほど必ず上がる。変数を増やすことでR²が上がっても、それが本当に有益かは「調整済みR²」で判断する。

#### Adj.R²（調整済み決定係数）

変数の数による過学習を補正したR²。**説明変数を追加したときに本当に改善したか**を判断するために使う。

| 変化 | 意味 |
|------|------|
| Adj.R²が上がった | その変数はモデルに貢献している |
| Adj.R²が下がった | その変数は不要（ノイズを増やしているだけ） |

**ECLモデルではAICやBICとセットで評価する。Adj.R²の目安は0.5以上。**

#### AIC（赤池情報量基準）

**モデルの複雑さを考慮した当てはまりの良さの指標。値が小さいほど良いモデル。**

- 複数のモデルを比較するときの主要指標
- 説明変数を増やすことのコスト（過学習ペナルティ）を考慮している
- AICが最小のモデルを「最良のモデル」の候補とする
- **絶対的な良し悪しはない。同じ目的変数・同じデータ期間のモデル間での相対比較にのみ使用する**

#### BIC（ベイズ情報量基準）

AICと同様だが、観測数が多いほど複雑なモデルにより大きなペナルティを課す。

| 比較 | 特徴 |
|------|------|
| AIC | 予測精度を重視。比較的多い変数を許容する |
| BIC | モデルの簡潔さを重視。より少ない変数を好む |

> **ECLモデルでの使い方**: 年次データで観測数が少ない場合（20〜30観測程度）は、AICとBICを両方確認し、両方で良好なモデルを採用する。観測数が少ないほどBICを重視する。

#### DW（Durbin-Watson統計量）

残差（モデルの予測誤差）に**自己相関（時系列的なパターン）**があるかを検定する指標。

| DW値 | 解釈 | アクション |
|------|------|-----------|
| 1.5〜2.5 | 自己相関なし（正常） | そのまま使用可能 |
| 2.5〜4.0 | 負の自己相関 | ラグ変数の追加を検討 |
| 0〜1.5 | **正の自己相関** | **ラグ変数の追加・ARIMA残差モデルの検討が必要** |

> **ECLモデルでの重要度**: 年度データは景気サイクルに連動するため、残差に正の自己相関（DW < 1.5）が出ることが多い。この場合、前年のPDをラグ変数として追加するか、残差をARIMAでモデル化することを検討する。

#### F統計量・F検定のp値

「全ての係数が0（モデルに説明力がない）」という帰無仮説を検定する。

| p値 | 解釈 |
|-----|------|
| 0.05未満 | モデル全体として有意。説明変数は集合的にPDに影響している |
| 0.05以上 | モデル全体として有意でない。変数の選び直しが必要 |

---

### 3.3 係数テーブルの読み方

OLS回帰の出力として、各変数（定数項 const を含む）について以下の値が得られる。

| 列名 | 意味 |
|------|------|
| **coef（係数）** | その変数が1単位増えたときの目的変数の変化量 |
| **std_err（標準誤差）** | 係数の推定の不確かさ |
| **t_stat（t統計量）** | `coef / std_err`。絶対値が大きいほど有意 |
| **p_value（p値）** | その変数が「真の係数=0（効果なし）」である確率 |

#### 係数（coef）の解釈

```
例: 失業率（unemployment_rate）の係数 = +0.0015

解釈（標準化なしの場合）:
  失業率が1%ポイント上昇すると、PDが0.0015（0.15%ポイント）上昇する

解釈（標準化ありの場合）:
  失業率が標準偏差1つ分上昇すると、PDが0.0015標準偏差分上昇する
  ※標準化した場合は係数の大小でどの変数が「より影響力が強いか」を比較できる
```

**経済学的な符号の確認が重要**:

| 変数 | 期待される符号 | 根拠 |
|------|-------------|------|
| 失業率 | **正（＋）** | 雇用悪化 → 返済能力低下 → PD上昇 |
| GDP成長率 | **負（−）** | 経済拡大 → 企業収益改善 → PD低下 |
| 金利 | **正（＋）** | 金利上昇 → 借入コスト増加 → PD上昇 |
| 地価指数 | **負（−）** | 担保価値上昇 → 担保余力改善 → LGD低下 |

> **係数の符号が経済学的直感と逆の場合**: 多重共線性の影響の可能性がある。VIFを再確認し、問題のある変数を除外した上で再推定すること。

#### p値の解釈

| p値 | 解釈 | アクション |
|-----|------|-----------|
| 0.01未満 | 1%水準で統計的に有意（強く有意） | 採用 |
| 0.01〜0.05 | 5%水準で統計的に有意 | 採用 |
| 0.05〜0.10 | 10%水準で有意（やや有意） | 理論的根拠があれば採用検討 |
| 0.10以上 | 統計的に有意でない | 除外を検討 |

> **ECLモデルでの注意**: 年次データは観測数が少ない（15〜30年分程度）ため、p値が0.05を上回ることは珍しくない。「理論的に重要な変数であれば、p < 0.10で採用する」という判断も合理的。重要なのは経済学的な整合性と複数指標の総合評価。

### 3.4 残差プロットの読み方

残差（= 実際の値 − モデルの予測値）は、モデルが捉えられなかった部分。

#### 残差 vs 予測値プロット

| 理想的な状態 | 問題がある状態 |
|------------|-------------|
| 予測値の大小に関わらずランダムに散らばっている | ∪字型・逆∪字型のパターンがある → 非線形関係が未捉 |
| 残差の幅が均一 | 予測値が大きいほど残差も大きくなる → 不均一分散（ヘテロスケダスティシティ） |

#### 時系列残差プロット

| 理想的な状態 | 問題がある状態 |
|------------|-------------|
| ランダムなノイズ | 持続する上昇・下降トレンド → 重要な変数の見落とし |
| 0近辺で上下 | 連続して正または負の残差が続く → 自己相関（DW < 1.5と連動） |
| 均等なばらつき | 特定の時期（例: リーマンショック、コロナ禍）に大きな残差 → 構造変化の可能性 |

#### 残差の正規性（Q-Qプロットまたはヒストグラム）

残差が正規分布に従うことはOLS回帰の仮定の1つ。

| 状態 | 解釈 | 対応 |
|------|------|------|
| 概ね直線上に乗っている | 正規分布に近い。問題なし | そのまま使用 |
| 両端が大きく乖離している | 外れ値または裾の重い分布 | 対数変換の検討 |
| 中央が膨らんでいる | 全体的に非正規 | 変換方法の見直し |

### 3.5 交差検証（k-fold CV）の結果

#### 出力内容

データをk個のグループに分けて、1つをテスト・残りを訓練として繰り返し予測精度を評価する。

| 指標 | 意味 |
|------|------|
| **平均MSE** | 各foldのMSE（二乗誤差の平均）を平均したもの。小さいほど良い |
| **MSE標準偏差** | foldによって予測精度がどれだけばらつくか。小さいほど安定している |
| **各foldのMSE** | foldごとの個別結果。特定foldだけ極端に悪ければ要因を調査 |

#### 解釈

```
例:
  平均MSE = 0.0002
  MSE標準偏差 = 0.00008

解釈:
  - PDの予測誤差の二乗平均が0.0002 → 標準誤差 ≒ √0.0002 ≒ 0.014（1.4%ポイント）
  - 各foldで安定した結果（標準偏差が平均の半分以下）
  - 過学習（訓練データのみでの高R²）が生じていないことを示す
```

> **ECLモデルでの使い方**: 年次データは観測数が少なく、5-fold CVでは各foldが3〜5年分しかない。交差検証の結果は参考程度とし、AIC/BICと経済学的整合性を優先する。

## 4. モデル選択タブ — 最適な変数の組み合わせを探す

### 4.1 このタブで行うこと

指定した数の説明変数候補のすべての組み合わせについて回帰を実行し、どの組み合わせが最も良いモデルかを自動的に探索する。

**例**: 10個の候補から2変数を選ぶ場合 → C(10,2) = 45通りを全て評価

### 4.2 結果テーブルの各列

| 列名 | 意味 | 良い値 |
|------|------|--------|
| **features** | 使用した説明変数の組み合わせ | — |
| **n_features** | 説明変数の数 | — |
| **r2** | 決定係数 | 大きいほど良い（目安: 0.5以上） |
| **adj_r2** | 調整済み決定係数 | 大きいほど良い（目安: 0.5以上） |
| **aic** | AIC | **小さいほど良い**（主要ソート基準） |
| **bic** | BIC | 小さいほど良い |
| **dw** | Durbin-Watson統計量 | 1.5〜2.5が理想 |
| **f_stat** | F統計量 | — |
| **f_pvalue** | F検定のp値 | 0.05未満が必要 |
| **max_vif** | 最大VIF（選択変数間） | **10以下が必須** |
| **nobs** | 有効観測数 | — |

### 4.3 モデル選択の手順

#### Step 1: VIFフィルタを適用する

まず `max_vif ≤ 10` でフィルタリングし、多重共線性がある組み合わせを除外する。

#### Step 2: AIC順の上位モデルを確認する

AICが小さい順に並んでいるので、上位5〜10件を「候補モデル」として記録する。

#### Step 3: 候補モデルを総合評価する

| チェック項目 | 確認内容 |
|------------|---------|
| f_pvalue | 0.05未満か？（モデル全体の有意性） |
| adj_r2 | 0.5以上か？（説明力） |
| dw | 1.5〜2.5の範囲か？（残差の自己相関） |
| max_vif | 10以下か？（多重共線性なし） |
| features | 経済学的に意味のある変数の組み合わせか？ |

#### Step 4: 上位候補を回帰分析タブで精査する

候補モデルを一つずつ回帰分析タブで実行し、各係数のp値と符号を確認する。

### 4.4 実務での解釈例

```
モデル選択結果（上位5件、2変数モデル）:

  rank | features                          | adj_r2 | aic    | dw   | max_vif
  1    | unemployment_rate, gdp_growth     | 0.72   | -45.3  | 1.87 | 2.1
  2    | unemployment_rate, ci_index       | 0.70   | -44.1  | 1.65 | 8.5
  3    | interest_rate, gdp_growth         | 0.68   | -43.8  | 1.43 | 1.8  ← DW警告
  4    | unemployment_rate, cpi_growth     | 0.65   | -42.5  | 1.92 | 1.9
  5    | land_price, gdp_growth            | 0.60   | -40.2  | 2.10 | 3.2

判断:
  - Rank1: AIC最小、adj_r2良好、DW正常、VIF低い → 最有力候補
  - Rank2: VIF=8.5で警告水準。2変数間の多重共線性を確認する
  - Rank3: DW=1.43で正の自己相関の懸念 → ラグ変数追加を検討
  - Rank4,5: adj_r2がRank1より低いが安定したモデル → 補足候補

→ Rank1を第一候補として回帰分析タブで係数を精査する
```

## 5. 動的回帰タブ — 変数別ラグの最適化

### 5.1 ラグとは何か

**ラグ（時差）**: マクロ経済指標の変化がPD/LGDに反映されるまでに時間がかかる現象。

```
例:
  2022年度の景気悪化（GDP低下）
    → 企業が資金繰りに詰まるのは2023年度〜2024年度
    → PDが上昇するのは2023年度〜2024年度

この場合、GDP成長率を1〜2年ラグで使用することが適切
```

### 5.2 ラグ設定の出力と解釈

各変数に異なるラグ（0〜n期）を設定して回帰を実行できる。

| 確認ポイント | 判断基準 |
|------------|---------|
| ラグを追加したときにAdj.R²が改善するか | 改善するならそのラグ次数を採用 |
| ラグを追加したときにDWが改善するか | 1.5〜2.5の範囲に近づくなら採用 |
| 係数の符号が経済学的に正しいか | 直感と逆の符号が出たら慎重に |
| AICが改善するか | ラグなしより小さければラグ有りを採用 |

### 5.3 ラグ選択の実務的な目安

| データの粒度 | 試すラグ範囲 | 経済学的根拠 |
|------------|-----------|------------|
| **年次（年度）** | 0〜3年ラグ | 景気サイクルは数年単位 |
| **四半期** | 0〜8四半期ラグ | 政策効果は2〜4四半期後 |
| **月次** | 0〜24ヶ月ラグ | 短期指標は1〜6ヶ月後 |

> **ECLモデルでの推奨**: まず「ラグなし」でベースモデルを作り、残差に自己相関（DW < 1.5）がある場合にラグ付きモデルを試す。ラグは理論的根拠（例: 「融資は審査から実行まで半年かかる」）と統計的改善の両方で判断する。

## 6. ARIMAタブ — マクロ経済指標の時系列予測

### 6.1 ARIMAとは

ARIMA(p, d, q) は時系列データを将来に外挿（延長）するためのモデル。

- **p（AR次数）**: 過去 p 期分の自分自身の値を使って予測する
- **d（差分次数）**: 定常化のために差分を取る回数
- **q（MA次数）**: 過去 q 期分の誤差を使って予測する

> **ECLモデルでの位置づけ**: 回帰モデルが確定したら、説明変数（GDP成長率、失業率等）の将来値をARIMAで予測する。その予測値を回帰式に代入することで、PD/LGDの将来予測値を計算する。

### 6.2 ADF検定（定常性検定）

#### 出力内容

| 項目 | 意味 |
|------|------|
| **ADF統計量** | 定常性の検定統計量。より負の値が定常性を示す |
| **p値** | 「非定常（単位根あり）」の帰無仮説の確率 |
| **臨界値（1%, 5%, 10%）** | 各水準での棄却域の境界値 |
| **is_stationary** | p値 < 0.05 の場合 True（定常と判断） |

#### 定常性とは

**定常系列**: 平均・分散・自己相関が時間によって変化しない系列。ARIMAモデルは基本的に定常系列を前提とする。

| 結果 | 解釈 | 次のアクション |
|------|------|-------------|
| **p値 < 0.05**（is_stationary = True） | 定常系列 | ARIMA(p, 0, q) を試す（d=0） |
| **p値 ≥ 0.05**（is_stationary = False） | 非定常系列（トレンドあり） | 差分化（d=1）して再検定 |
| 1次差分後も非定常 | 2次差分が必要な可能性 | d=2 でARIMAを試す |

```
例:
  系列: 失業率（2000年〜2025年）
  ADF統計量: -3.21
  p値: 0.021
  臨界値（5%）: -2.91

解釈:
  - p値 = 0.021 < 0.05 → 定常系列と判断
  - ADF統計量 (-3.21) < 5%臨界値 (-2.91) → 5%水準で帰無仮説を棄却
  - 差分化なし（d=0）でARIMAモデルを構築できる

例（非定常の場合）:
  系列: GDP（名目）
  ADF統計量: -0.85
  p値: 0.812

解釈:
  - p値 = 0.812 >> 0.05 → 非定常系列（トレンドがある）
  - 1次差分（=成長率）に変換してから再度ADF検定を行う
```

---

### 6.3 ACF・PACF プロット

#### ACF（自己相関関数）とは

**ラグ k 期前の値との相関係数**を可視化したもの。

| 形状 | 意味 |
|------|------|
| ラグ1以降で急速に0に近づく | AR成分が支配的 |
| 全ラグで緩やかに減衰する | 強いARプロセス（差分が必要かも） |
| 特定のラグで突出している | 季節性の可能性 |

#### PACF（偏自己相関関数）とは

**中間のラグの影響を除いた上での** ラグ k との相関係数。

#### ACF・PACFからのARIMA次数の読み取り

| ACFの形状 | PACFの形状 | 示唆されるモデル |
|----------|----------|--------------|
| q次で急激にカットオフ | 緩やかに減衰 | MA(q) モデル → ARIMA(0,d,q) |
| 緩やかに減衰 | p次で急激にカットオフ | AR(p) モデル → ARIMA(p,d,0) |
| 両方とも緩やかに減衰 | 両方とも緩やかに減衰 | ARMA(p,q) → ARIMA(p,d,q) |

> **実務での活用**: 次数の自動選択機能（後述）を使えば、ACF/PACFを詳細に分析しなくても最良次数を探索できる。ただし、ACF/PACFを確認することで「季節性があるか」「差分が適切だったか」を視覚的に判断できる。

**信頼区間（青い帯）の意味**: 帯の外に棒グラフが出ているラグは統計的に有意な自己相関が存在する。

---

### 6.4 次数自動選択の結果

#### 出力内容

全ての `(p, d, q)` の組み合わせを試し、AIC（またはBIC）が最小のモデルを選出する。

| 項目 | 意味 |
|------|------|
| **最良次数 ARIMA(p,d,q)** | AIC最小のモデルの次数 |
| **最良AIC/BIC** | 選出されたモデルの情報量基準 |
| **全結果一覧** | 全組み合わせのAIC・BIC・収束有無 |

#### 全結果一覧の見方

| 列 | 意味 |
|----|------|
| p, d, q | ARIMA次数 |
| aic | そのモデルのAIC（小さいほど良い） |
| bic | BIC |
| converged | True = 正常収束。False = 推定失敗 |

```
例:
  最良次数: ARIMA(1, 1, 0)
  AIC: 134.5
  BIC: 138.2

解釈:
  - p=1: 1期前の値を使って予測
  - d=1: 1次差分で定常化（原系列にトレンドあり）
  - q=0: MA成分なし
  - = 「前期比の変化が継続する傾向がある」シンプルなモデル
```

---

### 6.5 将来予測の結果

#### 出力内容

| 列 | 意味 |
|----|------|
| **forecast（予測値）** | モデルによる点予測値 |
| **lower（下限）** | 95%信頼区間の下限（デフォルトα=0.05） |
| **upper（上限）** | 95%信頼区間の上限 |

#### 解釈

**信頼区間（lower〜upper）**: 予測値の不確かさを表す範囲。

- 予測期間が長くなるほど信頼区間は広がる
- 信頼区間が非常に広い場合、その系列の将来予測は信頼性が低い
- ECLのシナリオ計算では、**予測値（中央値）をベースシナリオ、上限・下限を楽観/悲観シナリオの参考値として活用する**

```
例:
  時点      | forecast | lower  | upper
  2026Q1    | 2.8%     | 2.2%   | 3.4%
  2026Q2    | 2.9%     | 2.1%   | 3.7%
  2026Q3    | 3.0%     | 1.9%   | 4.1%  ← 期間が長くなるほど幅が広がる

ECLへの活用:
  ベースシナリオ: 失業率 = forecast (2.8%, 2.9%, 3.0%)
  楽観シナリオ:  失業率 = lower  (2.2%, 2.1%, 1.9%)
  悲観シナリオ:  失業率 = upper  (3.4%, 3.7%, 4.1%)
```

## 7. モデル確定タブ — 回帰モデルをDBに登録する（実装予定）

| 保存項目 | 内容 | ECLでの使い方 |

## 8. ECL計算タブ — 3シナリオでECLを算出する（実装予定）

### 8.1 操作の流れ

```
Step 1: 回帰モデルを選ぶ

Step 5: 「ECLを計算する」ボタンを押す
```

### 8.2 計算結果の見方

#### シナリオ別PD予測グラフ

| 線の色 | 内容 |
|--------|------|
| 青（実線） | ベースシナリオPD（ARIMA予測値をモデルに代入） |
| 緑（実線） | 楽観シナリオPD（ARIMA下限値をモデルに代入） |
| 赤（実線） | 悲観シナリオPD（ARIMA上限値をモデルに代入） |
| 黒（破線） | 加重平均PD（3シナリオをウェイトで平均した値） |

**グラフの読み方のポイント**:

| 着目点 | 解釈 |
|--------|------|
| ベースと悲観シナリオの乖離が大きい | 不確実性が高い予測期間。ウェイト設定の根拠を慎重に検討する |
| 加重平均PDがベースシナリオより上 | 悲観シナリオのウェイトが大きく、保守的な計算結果 |
| 3年目以降でシナリオの幅が急拡大 | ARIMAの信頼区間が広がっており、長期予測の不確実性が高い |

#### 結果テーブルの各列

| 列名 | 意味 | 利用用途 |
|------|------|---------|
| **期間** | 予測年度（例: 2026年度） | — |
| **PD_base** | ベースシナリオのPD予測値 | シナリオ比較 |
| **PD_upside** | 楽観シナリオのPD予測値 | シナリオ比較 |
| **PD_downside** | 悲観シナリオのPD予測値 | シナリオ比較 |
| **PD_weighted** | 加重平均PD | **引当金計算に使用** |
| **LGD_weighted** | 加重平均LGD（LGDモデルがある場合） | **引当金計算に使用** |
| **ECL率** | PD_weighted × LGD_weighted | 残高に掛けてECL金額を算出 |

```
計算例（2026年度）:
  PD_base     = 2.3%
  PD_upside   = 1.9%
  PD_downside = 3.1%
  ウェイト     = ベース60%, 楽観20%, 悲観20%

  PD_weighted = 0.60 × 2.3% + 0.20 × 1.9% + 0.20 × 3.1%
              = 1.38% + 0.38% + 0.62%
              = 2.38%

  LGD_weighted = 45.0%（LGDモデルが同様に計算済みと仮定）

  ECL率        = 2.38% × 45.0% = 1.07%

  EAD = 1,000億円 と入力した場合:
  ECL金額      = 1,000億円 × 1.07% = 10.7億円
```

### 8.3 シナリオウェイトの設定指針

| シナリオ | 典型的なウェイト | 根拠の例 |
|---------|--------------|---------|
| **ベース** | 50〜70% | 市場コンセンサス予測・日銀展望レポート等のベースラインに準拠 |
| **楽観** | 10〜25% | 設備投資増加・輸出回復等、景気回復が早まる場合 |
| **悲観** | 15〜30% | 海外景気後退・金融市場の混乱・自然災害等のテールリスク |

> **IFRS9の要求**: ウェイトの設定根拠を文書化し、毎期末に見直すことが求められる。「合理的で裏付けのある情報」に基づくことが監査上重要となる。ウェイトは統計的に決定される必要はなく、経営判断・マクロ見通しを踏まえた合理的な設定でよい。

### 8.4 ARIMAの信頼区間とシナリオの対応関係

ARIMAの予測結果（forecast/lower/upper）をそのままシナリオに使う場合、信頼区間の意味に注意する。

| ARIMA出力 | 意味 | シナリオへの割り当て |
|-----------|------|------------------|
| forecast（点予測） | 最も可能性が高い予測値 | ベースシナリオ |
| lower（下限） | 95%信頼区間の下限（楽観的な外れ値） | 楽観シナリオ |
| upper（上限） | 95%信頼区間の上限（悲観的な外れ値） | 悲観シナリオ |

> **注意**: ARIMAの信頼区間はあくまで統計的な不確実性の範囲。「失業率の下限 = 楽観シナリオ」「失業率の上限 = 悲観シナリオ」という対応が経済的に正しいかを変数ごとに確認すること。
> - 失業率: lower（低失業率）= 楽観、upper（高失業率）= 悲観 → **正しい**
> - GDP成長率: lower（低成長）= 悲観、upper（高成長）= 楽観 → **逆になるため注意**（実装時に変数ごとの正負を設定できるよう対応予定）

### 8.5 EADの扱い

EAD（デフォルト時エクスポージャー）は残高データから別途取得する。

| 入力方法 | 用途 |
|---------|------|
| 金額を入力（例: 1,000億円） | ECL金額（円）を計算。引当金額の確認に使用 |
| 省略（空欄） | ECL率（%）のみ計算。残高との掛け算は別途 |

> **本システムの位置づけ**: 本システムはPDおよびLGDのマクロ連動モデル構築・ECL率計算が主目的。EADの管理・計算は別の勘定システムで行い、ECL率を受け渡す運用を想定している。

### 8.6 モデルの限界と注意事項

| 限界 | 対応策 |
|------|--------|
| **学習データ期間外の出来事には対応できない** | コロナ禍・リーマンショック等は異常値として扱い、ダミー変数の追加や外れ値除外を検討する |
| **構造変化があるとモデルが陳腐化する** | 毎年モデルを再推定する。係数の変化をモニタリングし、大幅な変化があれば再選択を行う |
| **長期予測ほど不確実性が高くなる** | Stage2・Stage3（期待存続期間が長い）への適用では信頼区間の拡大を認識し、保守的なウェイト設定を検討する |
| **相関は因果関係ではない** | 係数の経済学的解釈を慎重に行い、「なぜその変数がPDに影響するか」の説明ロジックを整備する |
| **ARIMAは外的ショックを織り込まない** | 予測期間中に政策変更・地政学リスク等が想定される場合はシナリオを手動調整する |

| max_vif | 多重共線性 | < 5 | 5〜10 | > 10 |
| DW統計量 | 残差の自己相関 | 1.5〜2.5 | 1.2〜1.5 or 2.5〜2.8 | < 1.2 or > 2.8 |
| 係数のp値（全変数） | 変数の有意性 | < 0.05 | 0.05〜0.10 | > 0.10 |
| 残差の正規性 | OLS前提 | 概ね正規 | 軽度の歪み | 強い歪み |

### 7.2 経済学的観点

| チェック項目 | 確認内容 |
|------------|---------|
| 係数の符号 | 経済学的直感と一致しているか |
| 係数の大きさ | 現実的な感応度か（例: GDP1%低下でPDが10%増加は過大では？）|
| 選択した変数 | 実際のビジネス環境と関係している変数か |
| 予測の頑健性 | 過去データの一部を除いてもモデルが安定しているか |

### 7.3 実務的観点

| チェック項目 | 確認内容 |
|------------|---------|
| データの入手可能性 | 将来予測に使う説明変数の予測値が入手できるか |
| 更新頻度 | 説明変数の公表時期がECL計算のスケジュールに合っているか |
| 説明可能性 | 監査人・当局への説明が可能か（ブラックボックスでないか） |

## 10. ECL算出の全体フロー（参照用）

タブ①〜⑧を通じた年次作業の全体像を整理する。詳細は各セクションを参照。

### 10.1 分析から計算までの全体フロー

```
【データ更新】
① データ閲覧タブ
    最新の説明変数CSV・PD/LGD実績CSVをインポートする

【モデル探索（変数を変える年のみ）】
② 相関分析 → ④ モデル選択 → ③ 回帰分析 → ⑤ 動的回帰
    変数の入替・改善が必要な場合に実施する

【確定フェーズ（毎年実施）】
⑥ ARIMAタブ
    各説明変数について最新データでARIMAを再推定し、n年分の予測値をDBに保存する

⑦ モデル確定タブ（モデル改定時のみ）
    変数・設定を選択して回帰を実行し確認する
    「このモデルをDBに保存」でconfig_idを取得する
    変数を変えない年はスキップし、前年のconfig_idを流用する

【計算フェーズ（毎年実施）】
⑧ ECL計算タブ
    ─ 確定済み回帰モデル（config_id）を選択
    ─ 各説明変数にARIMA予測（forecast_id）を割り当てる
    ─ シナリオウェイトとEADを設定する
    ─ ECLを計算し、結果をDBに保存する
    ─ 必要に応じてCSVエクスポートし、引当金計算システムに引き渡す
```

### 10.2 ECL算式のまとめ

```
PD_weighted(t) = w_base × PD_base(t) + w_upside × PD_upside(t) + w_downside × PD_downside(t)

LGD_weighted(t) = w_base × LGD_base(t) + w_upside × LGD_upside(t) + w_downside × LGD_downside(t)

ECL率(t)       = PD_weighted(t) × LGD_weighted(t)

ECL金額(t)     = EAD(t) × ECL率(t)
```

> IFRS9では上記を全存続期間にわたって合計（または加重平均）し、割引効果を考慮した引当額を計算する。割引計算は本システムの対象外であり、別途引当金計算システムで処理する。

## 付録A: よくある問題と対処法

### A.1 回帰分析で「係数の符号が逆」になった

**原因**: 多重共線性の可能性が高い。

**対処**:
1. VIFを確認し、10以上の変数ペアを特定する
2. 相関係数が高いペアのどちらかを除外する
3. 再度回帰を実行して符号を確認する

### A.2 R²は高いが交差検証のMSEが大きい

**原因**: 過学習（訓練データへの過適合）。

**対処**:
1. 説明変数の数を減らす
2. 観測数に対して変数が多すぎないか確認する（経験則: 変数1個につき10観測以上）
3. より単純なモデルに変更する

### A.3 DWが小さく、残差に自己相関がある

**原因**: モデルが時系列的なパターンを捉えられていない。

**対処**:
1. 動的回帰タブで説明変数にラグを追加する
2. 目的変数のラグ値（例: 前年PD）を説明変数に追加する
3. ARIMAで残差をモデル化する（将来実装予定）

### A.4 ADF検定で「非定常」と判定された

**原因**: 系列に時間トレンドがある。

**対処**:
1. 差分化変換（`diff` または `log_diff`）を適用する
2. 差分化後に再度ADF検定を実行する
3. ARIMA の d パラメータに差分次数を指定する

### A.5 ARIMA次数の自動選択で「全てのモデルが失敗した」と表示される

**原因**: データが短すぎる、または欠損値がある。

**対処**:
1. データの長さを確認する（ARIMA(p,d,q) には最低 p+d+q+20 観測程度必要）
2. 欠損値を含む行を除外する
3. max_p, max_d, max_q を小さくして試す（例: p≤2, d≤1, q≤2）
