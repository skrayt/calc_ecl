予想信用損失モデルに基づく引当金計算プログラム作成の要件定義と開発手順

本ドキュメントは、企業会計基準公開草案第89号「金融商品に関する会計基準（案）」および関連する適用指針案に基づき、予想信用損失（ECL）モデルを用いた引当金計算プログラムを構築するための実務的な手順、データベース設計、およびプログラム仕様について詳述する。


--------------------------------------------------------------------------------


1. プログラム開発の全体概要

新たな引当金算定基準では、従来の「発生損失モデル」から、将来予測情報を織り込んだ「予想信用損失モデル」への転換が求められる。プログラムは、債権等の発生認識時からの信用リスクの増大（SICR）を判定し、それに応じて「12か月」または「全期間」の予想信用損失を算定する機能を備える必要がある。

開発のアプローチ

ソースコンテキストに基づき、以下の2つのアプローチを考慮した設計を行う。

* ステップ2（原則的アプローチ）: IFRS第9号と整合性を重視し、全期間のデフォルト確率（PD）の変動を厳密に計算する。
* ステップ4（簡素化アプローチ）: 実務負担に配慮し、内部信用格付の遷移や債務者区分を活用して判定・算定を行う。


--------------------------------------------------------------------------------


2. データベース設計要件

将来予想モデルを機能させるためには、過去の実績データ、現在の契約情報、および将来の外部経済指標を統合的に管理するデータベースが必要である。

2.1 契約・債権基本情報テーブル

債権等の発生時の情報を保持することは、信用リスクの「相対的変化」を測定するために不可欠である。

項目名	説明	備考
債権ID	契約ごとの一意識別子	
発生認識日	契約締結または実行日	SICR判定の起点
発生時信用格付	実行時点の内部格付・PD	相対判定に使用
当期末信用格付	決算時点の最新格付・PD	
実効金利（EIR）	発生時の実効金利または近似値	キャッシュ・フローの割引に使用
予想存続期間	契約上の最長期間（延長オプション含む）	全期間の算定に使用
担保・保証情報	担保の種類、評価額、回収見込時期	LGD（貸倒時損失率）の算出に使用

2.2 将来予測・外部指標テーブル

将来予想モデルの中核となる、合理的で裏付け可能な情報を格納する。

項目名	具体的なデータポイント（ソースに基づく例）
マクロ経済指標	国内総生産（GDP）、失業率、不動産価格、商品価格
市場指標	信用スプレッド、CDS価格、株価
地域・業種別情報	営業地域の特性、特定の産業構造に関連する指標
将来予測シナリオ	各指標の「中心的な予測」「楽観」「悲観」等の数値と発生確率

2.3 過去実績データテーブル

将来予測の調整根拠となるヒストリカルデータを保持する。

項目名	内容
貸倒実績率	格付別、業種別の過去のデフォルト実績
回収実績	担保処分による回収金額、回収期間
格付遷移マトリクス	一定期間内における格付の変動実績

3. プログラム仕様とロジック構成

プログラムは大きく「グルーピング」「信用リスクの著しい増大（SICR）の判定」「予想信用損失（ECL）の算定」の3つのモジュールで構成する。

3.1 信用リスクの著しい増大（SICR）判定ロジック

債権等を以下の3つのステージに振り分ける判定を行う。

1. 相対的判定（原則）:
  * 「期末のデフォルト発生リスク」と「発生認識時のデフォルト発生リスク」を比較する。
  * 全期間のPDの変動、またはその近似としての12か月PDの変動が一定の閾値を超えた場合にSICRが生じたとみなす。
2. 簡素化判定（内部格付活用）:
  * 正常先: 「優良格付」「中間格付」「要判定格付」に細分化。
  * 要注意先: 「その他要注意先」と「要管理先」に区分。
  * 格付が「要判定格付」以下に低下した場合、原則として全期間のECLを算定する。
3. 形式的判定（バックストップ）:
  * 30日（または1か月）超の延滞が発生している場合、SICRが生じていると推定する。

3.2 将来予想モデルの構築（最重要項目）

将来の経済状況の予測を引当金計算に反映させるためのプログラム仕様。

① 情報の反映プロセス

* 過去の貸倒実績データに対し、期末において観測可能な「現在の状況」および「将来の経済予測（GDP、失業率等）」に基づいた調整を加えるロジックを実装する。
* 過去の変動とマクロ経済指標の間に相関関係が見られる場合、その相関を反映した予測PDを算出する。

② 確率加重シナリオ計算

* 原則的処理: 複数の将来経済シナリオ（例：標準、悪化、大幅悪化）を設定し、それぞれのシナリオ下での信用損失額を計算した後、各シナリオの発生確率で加重平均を行う。
* 簡素化処理: 「最も可能性が高い中心的な将来予測シナリオ」のみを用いることができる。ただし、その場合でも「信用損失が発生しない可能性」と「発生する可能性」の両方を考慮した確率的リスクを織り込む。

③ キャッシュ・フローの見積り

* すべての契約条件（期限前償還、延長オプション等）を考慮し、予想存続期間にわたる期待キャッシュ・フローを算出する。
* 担保権の実行による回収見込額（金額および時期）を織り込む。


--------------------------------------------------------------------------------


4. 計算処理手順（ステップ・バイ・ステップ）

プログラムが実行すべき処理フローは以下の通りである。

1. データインポート: 契約情報、格付情報、マクロ経済予測指標をDBから読み込む。
2. グルーピング: 共通の信用リスク特性（業種、所在地、担保種類等）に基づき債権を集合的な単位に分類する。
3. SICR判定:
  * 各債権・グループについて、発生時と現在の信用リスクを比較。
  * 12か月ECL（ステージ1）か全期間ECL（ステージ2・3）かを決定。
4. 将来予測パラメータの適用:
  * 設定された経済シナリオに基づき、PD（デフォルト確率）およびLGD（貸倒時損失率）を将来予測値に補正する。
5. キャッシュ・フロー不足額の算出:
  * 「契約上のキャッシュ・フロー」と「将来予測に基づく期待キャッシュ・フロー」の差額を算出。
6. 時間価値の割引処理:
  * 算出された不足額を、発生時の実効金利（EIR）を用いて期末時点まで割り引く。
7. 引当金の確定: 確率加重された割引後不足額を合計し、貸倒引当金として計上する。


--------------------------------------------------------------------------------


5. 留意すべき特殊なプログラム仕様

* 金融保証契約・貸出コミットメント: 実行済みの債権だけでなく、未使用枠や保証債務に対しても、実行（デフォルト）の確率を考慮した予想信用損失を算定する機能を備える。
* 直接減額: 回収の合理的な予想を有していない金額について、貸倒引当金と相殺して帳簿価額を直接減額する処理ロジックを組み込む。
* 開示用データの生成: 注記に必要な「予想信用損失引当金の期首残高から期末残高への調整表」を作成するための、変動要因別（設定、戻入、直接減額等）の集計機能を備える。
